Step 1: SCSS, Pug и JS-генерация разметки

Исходник таймера взят из CodePen Nicolas Jesenberger. В CodePen автор использует сразу два препроцессора: SCSS для стилей и Pug для HTML. В проекте оставляем SCSS, а Pug не подключаем: повторяющуюся разметку шкалы создаем через JavaScript.

Рабочая договоренность по именам в этом проекте: используем number/numbers вместо исходного face/faces, чтобы понятнее читать шкалу как набор числовых сегментов.

Что уже выяснили

  • Браузер напрямую читает index.html, style.css и script.js.
  • SCSS браузер напрямую не читает. Его нужно компилировать в обычный CSS.
  • Pug-строки из CodePen нельзя вставлять прямо в обычный HTML: браузер их не выполнит.
  • Для 48 повторяющихся сегментов шкалы не пишем HTML руками, а генерируем его через JS.

SCSS в этом компоненте

Sass установлен локально внутри папки components/functional_egg_timer. Все команды ниже выполняются именно из этой папки.

Команды для Sass 
cd components/functional_egg_timer
npm install -D sass

npm list sass
npx sass --version

npx sass style.scss style.css
npx sass --watch style.scss:style.css
Правило работы: пишем в style.scss, а HTML подключает style.css. После запуска Sass файл style.css лучше не редактировать руками, потому что он будет перезаписываться.

Source map и Git

Sass может создавать style.css.map. Это sourcemap-файл для DevTools: он помогает браузеру связывать скомпилированный CSS с исходным SCSS.

Для учебного проекта его можно не хранить в репозитории. Корневой .gitignore работает на весь проект и все вложенные папки, поэтому отдельный .gitignore внутри таймера не нужен.

Ignore CSS source maps 
# .gitignore
*.css.map
Если style.css.map уже был закоммичен, .gitignore сам его из Git не удалит. Нужно один раз убрать файл из индекса через git rm --cached.

Почему Pug-блок автора не работал

В CodePen этот блок сначала компилируется Pug-ом в обычный HTML. В нашем статическом index.html такой синтаксис не выполняется.

Pug-код из CodePen, переименованный под number/numbers 
- const numberCount = 48
.egg-top(style=`--number-count: ${numberCount}`)
    .time-ruler-numbers-container
        - for (let i = 0; i < numberCount; i++)
            .time-ruler-number-wrapper(style=`--i: ${i}`)
                svg.time-number-face(xmlns="http://www.w3.org/2000/svg" viewBox="0 0 1740 76", aria-hidden="true")
                    use(href="#time-ruler")

Какую структуру должен получить браузер

В обычном DOM должна появиться такая вложенность. Внутри контейнера создаются 48 .time-ruler-number-wrapper с индексами от 0 до 47.

Итоговая DOM-структура 
<div class="egg-top" style="--number-count: 48">
    <div class="time-ruler-numbers-container">
        <div class="time-ruler-number-wrapper" style="--i: 0">...</div>
        <div class="time-ruler-number-wrapper" style="--i: 1">...</div>
        <div class="time-ruler-number-wrapper" style="--i: 2">...</div>
    </div>
</div>

JS-аналог Pug-цикла

JS здесь не отвечает за 3D-эффект. Он только создает элементы и прокидывает числовые значения в CSS через custom properties.

Базовая генерация разметки 
document.addEventListener("DOMContentLoaded", () => {
    const numberCount = 48;

    const egg = document.querySelector(".egg");
    const eggCenter = document.querySelector(".egg-center");

    if (!egg || !eggCenter) {
        return;
    }

    const eggTop = document.createElement("div");
    eggTop.className = "egg-top";
    eggTop.style.setProperty("--number-count", numberCount);

    const numbersContainer = document.createElement("div");
    numbersContainer.className = "time-ruler-numbers-container";

    const numberHtml = Array.from({ length: numberCount }, (_, index) => {
        return `
            <div class="time-ruler-number-wrapper" style="--i: ${index}">
                <svg class="time-number-face" xmlns="http://www.w3.org/2000/svg" viewBox="0 0 1740 76" aria-hidden="true">
                    <use href="#time-ruler"></use>
                </svg>
            </div>
        `;
    });

    numbersContainer.innerHTML = numberHtml.join("");
    eggTop.append(numbersContainer);
    egg.insertBefore(eggTop, eggCenter);
});

Итог Step 1

  1. SCSS оставляем как учебный слой для стилей.
  2. Pug не подключаем, потому что это был бы второй препроцессор для HTML.
  3. Повторяющуюся разметку шкалы создаем через JavaScript.
  4. В DOM должна появиться структура egg-top -> numbers-container -> 48 wrappers -> svg.

Step 2: Семантика SCSS, CSS-переменные и геометрия шкалы

Этот шаг начался после паузы в проекте: стили не отображались, Sass записал ошибку прямо в style.css, а после компиляции стало видно, что шкала работает не как в CodePen. Разбор ниже фиксирует не только исправления, но и механику, почему этот таймер интересен.

Таймер логически простой, а сложность компонента в другом: JS создает 48 DOM-сегментов, SCSS считает геометрию, а CSS раскладывает куски SVG по 3D-цилиндру.

Проблемы компиляции Sass

Первая поломка выглядела как отсутствие стилей, но на деле style.css содержал CSS с текстом ошибки Sass через body::before.

Типичные Sass-ошибки из этого этапа 
Error: Undefined variable.
$face-count

Error: Only 1 argument allowed, but 2 were passed.
math.sin(math.$pi, $faceCount)

Error: Missing argument $number2.
math.div((math.$pi, $faceCount))
  • $faceCount или $numberCount - SCSS-переменная, она должна быть объявлена до использования.
  • math.sin() принимает один аргумент: уже готовое число или выражение.
  • math.div() принимает два отдельных аргумента: math.div(math.$pi, $faceCount).
  • Запись math.div((math.$pi, $faceCount)) ошибочна: внутренние скобки превращают два значения в один список.
Правильный Sass-модуль math 
@use "sass:math";

$faceCount: 48;

math.sin(math.div(math.$pi, $faceCount));
math.tan(math.div(math.$pi, $faceCount));

Три слоя переменных

Здесь важно не смешивать JS, CSS и SCSS. Они живут в разное время.

JS-время

numberCount и index существуют во время работы скрипта. JS создает элементы и записывает inline CSS-переменные.

CSS-время

--number-count и --i читаются браузером в var(...) и calc(...).

SCSS-время

$faceCount нужен Sass при компиляции. Sass заранее считает sin/tan и пишет готовые числа в CSS.

Нельзя смешивать

Sass не может взять значение из var(--number-count), потому что CSS-переменная появится только в браузере, уже после компиляции.

Подробно про Array.from

Строка ниже создает массив из 48 строк HTML. Источник массива - объект с полем length, а второй аргумент - функция для каждого будущего элемента.

Array.from structure 
const numberHtml = Array.from({ length: numberCount }, (_, index) => {
    return `...`;
});
  • { length: numberCount } - объект. Если numberCount = 48, то это как { length: 48 }.
  • Array.from(...) создает массив длиной 48.
  • Callback получает два стандартных аргумента: (currentValue, index).
  • В этом случае currentValue будет undefined, потому что массив создается только по длине.
  • _ - обычное имя параметра, не магия языка. Его используют как знак "этот аргумент не нужен".
  • index - второй параметр callback-а. JS передает туда индекс: 0, 1, 2 ... 47.
Имена параметров можно менять 
Array.from({ length: 3 }, (value, index) => {
    console.log(value, index);
});
// undefined 0
// undefined 1
// undefined 2

Array.from({ length: 3 }, (_, currentIndex) => {
    return currentIndex;
});
// 0, 1, 2

Зачем нужен --i

Каждый wrapper получает свой номер в inline style. Потом CSS использует этот номер для угла и для выбора своего фрагмента длинной SVG-линейки.

Индекс сегмента 
<div class="time-ruler-number-wrapper" style="--i: 0">...</div>
<div class="time-ruler-number-wrapper" style="--i: 1">...</div>
<div class="time-ruler-number-wrapper" style="--i: 2">...</div>

Почему угол умножается на --i

Контейнер содержит 48 сегментов. Полный круг - 360deg. Шаг между соседними сегментами:

Угол каждого сегмента 
--angle: calc(360deg / var(--number-count) * var(--i));

360deg / 48 = 7.5deg

i = 0  -> 7.5deg * 0  = 0deg
i = 1  -> 7.5deg * 1  = 7.5deg
i = 2  -> 7.5deg * 2  = 15deg
i = 47 -> 7.5deg * 47 = 352.5deg

Без умножения на var(--i) все сегменты получили бы один и тот же угол и легли бы в одно место.

Одна длинная SVG и 48 маленьких окон

Шкала не состоит из 48 разных картинок. Есть одна длинная SVG-линейка #time-ruler, а каждый wrapper показывает только свой кусок.

Wrapper как окно 
.time-ruler-number-wrapper {
    width: ...;
    overflow: hidden;
}

.time-number-face {
    left: calc(-100% * var(--i));
    width: auto;
    max-width: none;
}
  • overflow: hidden превращает wrapper в маленькое окно.
  • left: 0 показывает первый кусок SVG.
  • left: -100% показывает следующий кусок.
  • left: -200% показывает третий кусок.
  • max-width: none нужен, потому что общий CSS проекта сжимает SVG через svg { max-width: 100%; }.

Почему были длинная середина и широкая кнопка

Если .egg или .egg-container остаются блочными элементами на всю ширину карточки, то .egg-center с width: calc(100% - 24px) тоже растягивается на всю карточку. Кнопка с width: 100% ведет себя так же.

Ограничение ширины яйца 
.egg-container {
    width: 274px;
    margin: 0 auto;
}

.egg {
    width: 274px;
}

Геометрия: ширина грани

Сегменты шкалы раскладываются как стороны правильного многоугольника. При 48 сторонах он почти выглядит как цилиндр.

Ширина одного сегмента 
width: calc(100% * #{math.sin(math.div(math.$pi, $faceCount))});
  • math.$pi - число π, примерно 3.14159.
  • math.div(math.$pi, $faceCount) при 48 сегментах дает π / 48.
  • sin(π / 48) примерно равен 0.0654.
  • Итог для CSS: width: calc(100% * 0.0654).

В геометрии правильного многоугольника ширина стороны считается через синус половины угла сектора. Здесь 100% можно воспринимать как диаметр шкалы, а результат - как ширину одной грани.

Геометрия: расстояние от центра

Второй расчет отвечает за то, насколько далеко от центра нужно отодвинуть каждый сегмент, чтобы все 48 кусочков собрались в цилиндр.

Отступ от центра цилиндра 
translateX(
    calc(100% / (2 * #{math.tan(math.div(math.$pi, $faceCount))}) - #{$anti-gap-shift})
)
  • tan(π / 48) примерно равен 0.0655.
  • 100% внутри transform считается от размера самого wrapper-а.
  • Формула похожа на apothem = side / (2 * tan(π / N)).
  • $anti-gap-shift чуть сдвигает грань внутрь, чтобы между сегментами не было щелей.
Коротко: sin отвечает за ширину грани, а tan отвечает за расстояние грани от центра.

3D-контейнер и transform

Контейнер шкалы не является яйцом. Это держатель для 48 граней. Он сохраняет дочерние элементы в 3D-пространстве.

3D-контейнер 
.time-ruler-numbers-container {
    transform-style: preserve-3d;
}

preserve-3d говорит браузеру: не сплющивай детей контейнера в одну 2D-плоскость, оставь их повороты и смещения в 3D.

Раскладка одной грани 
transform:
    rotateY(calc(var(--angle) - 90deg))
    translateX(...)
    rotateY(90deg);
  • rotateY(calc(var(--angle) - 90deg)) поворачивает систему координат для конкретной грани.
  • translateX(...) отодвигает грань от центра цилиндра.
  • rotateY(90deg) разворачивает саму плоскость грани лицом наружу.
  • Порядок transform-функций важен: если поменять их местами, элемент окажется в другом месте.

Куб и цилиндр - одна идея

Через 3D transforms можно сделать и куб, и цилиндр. Куб - это 6 граней, цилиндр здесь - 48 граней. Чем больше граней, тем круглее выглядит форма.

Модель в одну строку 
6 граней  -> куб / шестигранник
48 граней -> почти круглый цилиндр

Что проверять в DevTools

  • Отключить overflow: hidden - станет видно длинную SVG внутри каждого окна.
  • Отключить left: calc(-100% * var(--i)) - все окна начнут показывать один и тот же кусок SVG.
  • Отключить transform у wrapper-а - грани перестанут собираться в цилиндр.
  • Проверить DOM: внутри .time-ruler-numbers-container должно быть 48 .time-ruler-number-wrapper.

Итог Step 2

  1. JS создает 48 сегментов и каждому задает --i.
  2. CSS по --i считает угол каждого сегмента.
  3. Каждый wrapper показывает свой кусок длинной SVG-линейки.
  4. SCSS через sin/tan считает размеры и радиус раскладки.
  5. 3D transform раскладывает сегменты по цилиндру.
  6. Будущее вращение шкалы будет происходить не через пересоздание DOM, а через изменение --rotation.

Step 3: Базовая JS-логика таймера

После визуальной части и 3D-шкалы перешли к логике таймера. Главная цель этого этапа: не сразу написать весь таймер одним куском, а собрать его из маленьких понятных функций. В конце этапа таймер уже запускается, тикает каждую секунду, доходит до нуля и останавливается.

В коде для практики используем формат TASK 01, TASK 02 и так далее. Для творческих ответвлений используем EXTRATASK: маленький эксперимент, после которого возвращаемся к основному маршруту.

С чего стартовала логика

Сначала JS должен был получить ссылки на элементы, которыми он будет управлять: контейнер таймера, яйцо, место вывода времени и кнопку.

DOM-ссылки 
const egg = document.querySelector(".egg");
const eggCenter = document.querySelector(".egg-center");
const eggContainer = document.querySelector(".egg-container");
const timeContainer = document.querySelector(".time-container");
const timerButton = document.querySelector(".timer-button");

После этого добавили защитную проверку. Если какой-то важный элемент не найден, выполнение останавливается через return.

Guard-проверка 
if (!egg || !eggCenter || !eggContainer || !timeContainer || !timerButton) {
    return;
}
  • return; здесь нужен не для возврата значения, а для остановки функции.
  • return null; тоже остановил бы функцию, но значение null никто не использует.
  • querySelector(...) возвращает один первый элемент или null.
  • querySelectorAll(...) возвращает список элементов, поэтому у списка нельзя напрямую менять textContent.

Состояние таймера

Для логики завели переменные состояния. Это данные, которые JS будет помнить и менять во время работы.

State-переменные 
let selectedSeconds = 0;
let remainingSeconds = 0;
let timerId = null;
let isRunning = false;

selectedSeconds

Сколько секунд выбрал пользователь. Это значение отвечает за возможность старта: если выбрано 0, кнопку нельзя включать.

remainingSeconds

Сколько секунд осталось прямо сейчас. Это значение меняется во время обратного отсчета: 60, 59, 58...

timerId

Коробка для ID интервала. Сначала там null, потому что активного интервала еще нет. После setInterval туда кладется ID.

isRunning

Флаг состояния: таймер сейчас запущен или нет. По нему кнопка понимает, вызывать startTimer() или stopTimer().

Форматирование времени

Внутри логики время хранится числом секунд. На экран нужно выводить строку формата 0:00, поэтому появилась отдельная функция formatTime.

formatTime 
function formatTime(totalSeconds) {
    const minutes = Math.floor(totalSeconds / 60);
    const seconds = totalSeconds % 60;
    const paddedSeconds = String(seconds).padStart(2, "0");

    return `${minutes}:${paddedSeconds}`;
}
  • Math.floor(totalSeconds / 60) получает целые минуты.
  • totalSeconds % 60 получает остаток секунд после полных минут.
  • String(seconds) превращает число в строку.
  • padStart(2, "0") делает 5 строкой "05".

textContent и innerHTML

Для вывода времени и текста кнопки используем textContent, потому что нам нужен обычный текст, а не новая HTML-разметка.

textContent vs innerHTML 
timeContainer.textContent = "1:05";
timerButton.textContent = "Stop";

// Вставит именно текст:
element.textContent = "<strong>Hello</strong>";

// Вставит HTML-разметку:
element.innerHTML = "<strong>Hello</strong>";

innerHTML оставляем для случаев, где действительно нужна разметка, например для генерации 48 SVG-wrapper-ов шкалы.

Разделение функций

Вместо того чтобы менять все прямо в обработчике клика, логику разделили на маленькие функции. Каждая отвечает за одну понятную вещь.

Обновление экрана и установка времени 
function updateTimeDisplay() {
    timeContainer.textContent = formatTime(remainingSeconds);
}

function setTimerTime(seconds) {
    selectedSeconds = seconds;
    remainingSeconds = seconds;
    updateTimeDisplay();
    updateButtonState();
}

function updateButtonState() {
    timerButton.disabled = selectedSeconds === 0;
}
Важное разделение: selectedSeconds отвечает за то, можно ли стартовать, а remainingSeconds отвечает за ход обратного отсчета.

Кнопка Start / Stop

Кнопка работает как переключатель. Если таймер уже запущен, клик останавливает его. Если не запущен - стартует.

Переключатель по isRunning 
timerButton.addEventListener("click", () => {
    if (isRunning) {
        stopTimer();
    } else {
        startTimer();
    }
});
startTimer / stopTimer 
function startTimer() {
    isRunning = true;
    timerButton.textContent = "Stop";
    timerButton.classList.add("stop");

    timerId = setInterval(() => {
        tickTimer();
    }, 1000);
}

function stopTimer() {
    isRunning = false;
    timerButton.textContent = "Start";
    timerButton.classList.remove("stop");

    clearInterval(timerId);
    timerId = null;
}
  • classList.add("stop") добавляет класс без точки.
  • Точка нужна в CSS-селекторе: .timer-button.stop.
  • classList.remove("stop") убирает класс и возвращает кнопку к обычному виду.

Один тик таймера

Перед автоматическим таймером сначала сделали функцию одного шага. Это помогло проверить механику без setInterval.

tickTimer 
function tickTimer() {
    if (remainingSeconds === 0) {
        stopTimer();
        return;
    }

    remainingSeconds = remainingSeconds - 1;
    updateTimeDisplay();
}

return внутри проверки нужен, чтобы функция не пошла дальше и не сделала remainingSeconds = -1.

EXTRATASK: лог времени завершения

Для эксперимента добавили лог в момент, когда таймер дошел до нуля. Это оказалось полезно: по логам стало видно, что setInterval продолжает работать, если его не остановить через clearInterval.

Лог момента завершения 
const logText = "CALL!!!";
const callDate = new Date();

console.log(`${logText} ${callDate.toLocaleTimeString()}`);
  • new Date() создает снимок текущего момента.
  • toLocaleTimeString() выводит локальное время, например 18:47:23.
  • setDate() не подходит для строки: это метод объекта Date, а не текста.

setInterval, clearInterval и timerId

setInterval запускает функцию снова и снова через заданный промежуток. В нашем случае он вызывает tickTimer() каждую секунду.

Запуск интервала 
timerId = setInterval(() => {
    tickTimer();
}, 1000);
  • 1000 - это 1000 миллисекунд, то есть 1 секунда.
  • setInterval возвращает ID интервала.
  • Этот ID сохраняем в timerId, чтобы потом остановить именно этот интервал.
  • clearInterval(timerId) останавливает интервал по сохраненному ID.
  • После остановки полезно сделать timerId = null: активного интервала больше нет.
Если не вызвать clearInterval(timerId), интервал продолжит работать даже после визуальной остановки кнопки. Это хорошо стало видно именно благодаря логам из EXTRATASK.

Текущий итог логики

  1. Таймер хранит состояние в selectedSeconds, remainingSeconds, timerId, isRunning.
  2. Кнопка включается только если выбрано время больше нуля.
  3. Start меняет состояние, кнопку и запускает интервал.
  4. Stop меняет состояние обратно и останавливает интервал.
  5. tickTimer() уменьшает оставшееся время и обновляет экран.
  6. Когда таймер доходит до 0, он останавливается и пишет в консоль время завершения.

Step 4: Поворот, 60 секунд, состояние Call и звуки

На этом шаге таймер стал почти полноценным: шкала начала управлять временем, обратный отсчет начал синхронно крутить циферблат назад, появилось отдельное состояние звонка и подключились звуки из оригинального CodePen.

Важная развилка проекта: оригинальный таймер рассчитан на 60 минут, а в этой учебной версии делаем максимум 60 секунд. Поэтому формулы пересчета поворота отличаются от оригинала.

Поворот шкалы и выбранное время

Мы храним поворот в переменной currentRotation. Это обычное число в градусах. Для нашей версии правило простое:

Модель 60 секунд 
360 градусов = 60 секунд
6 градусов = 1 секунда

0 градусов = 0 секунд
30 градусов = 5 секунд
180 градусов = 30 секунд
360 градусов = 60 секунд

Поэтому при ручной прокрутке делаем перевод из градусов в секунды через деление на 6.

rotation -> seconds 
function updateRotation(rotation) {
    currentRotation = clamp(rotation, 0, 360);
    renderRotation();

    const seconds = Math.round(currentRotation / 6);

    setTimerTime(seconds);
    egg.classList.remove("ringing");
}
  • clamp(rotation, 0, 360) не дает выйти ниже 0 и выше 60 секунд.
  • currentRotation / 6 переводит градусы в секунды.
  • Math.round округляет значение до ближайшей целой секунды.
  • setTimerTime(seconds) записывает выбранное время в selectedSeconds и remainingSeconds.

Обратная синхронизация: время обратно в поворот

Когда таймер уже запущен, он каждую секунду уменьшает remainingSeconds. Но одной цифры на экране мало: шкала тоже должна двигаться назад. Для этого нужна обратная формула.

seconds -> rotation 
function syncRotationToTime() {
    currentRotation = remainingSeconds * 6;
    renderRotation();
}

Здесь уже не делим, а умножаем. Если осталось 10 секунд, шкала должна быть на 60deg. Если осталось 60 секунд, шкала должна быть на 360deg.

Проверка формулы 
4 секунды * 6 = 24 градуса
30 секунд * 6 = 180 градусов
54 секунды * 6 = 324 градуса
60 секунд * 6 = 360 градусов
Ошибка, из-за которой шкала двигалась только внутри одного деления: была старая формула remainingSeconds / 10. Для 60 секунд она давала 60 / 10 = 6deg, то есть весь таймер сжимался до одного маленького деления.

Почему в CSS уходит минус

В коде храним currentRotation как положительное число: 0..360. Но визуально шкалу нужно повернуть в другую сторону, поэтому в CSS-переменную отправляем значение с минусом.

renderRotation 
function renderRotation() {
    numbersContainer.style.setProperty("--rotation", `${-currentRotation}deg`);
}

Это разделяет две вещи: в JS удобная модель 0..360, а в CSS нужное направление визуального поворота.

Pointer-события и ручное вращение

Для прокрутки шкалы используются pointer-события. Они работают для мыши, пальца и стилуса. Главная идея: запомнить точку старта и посчитать, насколько пользователь сдвинул указатель.

deltaX и новый поворот 
dragStartX = event.clientX;
dragStartRotation = currentRotation;

const deltaX = event.clientX - dragStartX;
const rotation = dragStartRotation - deltaX;

updateRotation(rotation);
  • event.clientX - координата указателя по горизонтали в окне браузера.
  • dragStartX - где началось перетаскивание.
  • deltaX - насколько указатель ушел от стартовой точки.
  • dragStartRotation - какой поворот был в момент начала движения.
  • dragStartRotation - deltaX дает ощущение нужного направления вращения.

setPointerCapture(event.pointerId) закрепляет pointer-событие за элементом. Это нужно, чтобы движение не обрывалось, если указатель вышел за пределы верхней части яйца. После окончания движения вызываем releasePointerCapture(event.pointerId).

Отдельное состояние звонка: isCalling

isRunning отвечает только за обратный отсчет. Когда таймер дошел до нуля, отсчет уже не идет, но приложение еще находится в состоянии звонка. Поэтому нужен второй флаг: isCalling.

Два разных состояния 
let isRunning = false;
let isCalling = false;
Клик по кнопке 
timerButton.addEventListener("click", () => {
    if (isCalling) {
        stopCall();
        return;
    }

    if (isRunning) {
        stopTimer();
    } else {
        startTimer();
    }
});

Проверка isCalling должна идти первой. Если таймер звонит, следующий клик должен выключить звонок, а не запускать новый отсчет.

Финиш таймера и выключение Call

Когда remainingSeconds дошел до нуля, обычный интервал нужно остановить, а потом включить состояние звонка.

finishTimer 
function finishTimer() {
    stopTimer();
    isCalling = true;

    timerButton.textContent = "Stop call";
    timerButton.classList.add("stop");
    timerButton.disabled = false;

    timerAlarmSound.play();
    egg.classList.add("ringing");
}
stopCall 
function stopCall() {
    isCalling = false;
    egg.classList.remove("ringing");
    timerAlarmSound.stop();

    timerButton.textContent = "Start";
    timerButton.classList.remove("stop");
    updateButtonState();
}
  • stopTimer() останавливает обычный отсчет и тиканье.
  • finishTimer() включает состояние звонка.
  • stopCall() выключает звонок и возвращает кнопку в обычное состояние.
  • timerButton.disabled = false нужен, чтобы кнопку можно было нажать даже при нуле.

Howler и Howl

Howl - не встроенный объект JavaScript. Он появляется только после подключения библиотеки Howler. Поэтому скрипт библиотеки должен стоять выше нашего script.js.

Подключение Howler 
<script src="https://cdnjs.cloudflare.com/ajax/libs/howler/2.2.4/howler.min.js"></script>
<script src="./script.js"></script>

Браузер читает HTML сверху вниз. Если сначала подключить наш файл, а уже потом Howler, то new Howl(...) упадет с ошибкой Howl is not defined.

Три звука таймера 
const timerWindingSound = new Howl({
    src: ["https://assets.codepen.io/4175254/timer-winding_1.mp3"],
});

const timerAlarmSound = new Howl({
    src: ["https://assets.codepen.io/4175254/timer-alarm.mp3"],
});

const timerTickingSound = new Howl({
    src: ["https://assets.codepen.io/4175254/timer-ticking.mp3"],
    loop: true,
});
  • timerWindingSound - щелчки при ручной прокрутке шкалы.
  • timerTickingSound - фоновое тиканье во время запущенного отсчета.
  • timerAlarmSound - звонок, когда таймер дошел до нуля.
  • loop: true значит, что звук будет повторяться по кругу, пока его не остановить.

Почему звук по делениям - это Winding, а не Ticking

В оригинале щелчки при ручной прокрутке делает не timerTickingSound. Их делает timerWindingSound. Это видно по условию, где проверяется переход на новое деление.

Идея из оригинала 
if (Math.abs(adjustedRotation - lastRotation) >= 6) {
    timerWindingSound.play();
    navigator.vibrate?.(50);
    lastRotation = adjustedRotation;
}

Число 6 здесь не случайное. В исходной шкале 6 градусов - это одно крупное деление. В нашей версии 6 градусов - это одна секунда.

Разбор playWindingSoundOnSecondChange()

Эта функция нужна, чтобы звук срабатывал не на каждый пиксель движения, а только когда выбранная секунда реально изменилась.

Звук при переходе на новое деление 
let lastWindingSecond = 0;

function playWindingSoundOnSecondChange() {
    const currentSecond = Math.round(currentRotation / 6);

    if (currentSecond === lastWindingSecond) {
        return;
    }

    timerWindingSound.play();
    navigator.vibrate?.(50);
    lastWindingSecond = currentSecond;
}

Построчно:

  • lastWindingSecond хранит последнюю секунду, на которой звук уже сыграл.
  • currentRotation / 6 переводит текущий поворот в секунды.
  • Math.round(...) округляет результат до ближайшей секунды.
  • Если currentSecond === lastWindingSecond, значит мы все еще внутри того же деления, звук не нужен.
  • return досрочно выходит из функции.
  • timerWindingSound.play() проигрывает щелчок.
  • navigator.vibrate?.(50) включает вибрацию на 50 миллисекунд, если устройство поддерживает вибрацию.
  • lastWindingSecond = currentSecond запоминает новое озвученное деление.
?. называется optional chaining. В выражении navigator.vibrate?.(50) оно значит: если метод vibrate существует, вызови его; если не существует, просто ничего не делай и не падай с ошибкой.

Где вызывается звук по делениям

Начальное деление запоминается в момент pointerdown. Потом при каждом pointermove сначала обновляется поворот, а затем проверяется, перешли ли мы на новую секунду.

pointerdown / pointermove 
eggTop.addEventListener("pointerdown", (event) => {
    isDragging = true;
    lastWindingSecond = Math.round(currentRotation / 6);
    dragStartX = event.clientX;
    dragStartRotation = currentRotation;
});

eggTop.addEventListener("pointermove", (event) => {
    if (!isDragging) {
        return;
    }

    const deltaX = event.clientX - dragStartX;
    const rotation = dragStartRotation - deltaX;

    updateRotation(rotation);
    playWindingSoundOnSecondChange();
});

Итог Step 4

  1. Шкала теперь работает как таймер на 60 секунд, а не на 60 минут.
  2. Поворот в градусах переводится в секунды через currentRotation / 6.
  3. Оставшееся время переводится обратно в поворот через remainingSeconds * 6.
  4. Состояние isCalling отделяет звонок от обычного запущенного таймера.
  5. Howler подключается отдельным скриптом до script.js.
  6. timerTickingSound играет во время отсчета, timerWindingSound щелкает при прокрутке, timerAlarmSound звонит на нуле.
  7. Звук прокрутки должен срабатывать при смене деления, а не на каждый пиксель движения.
