Step 1: SCSS, Pug и генерация разметки через JS

Исходник таймера взят из CodePen Nicolas Jesenberger. Главная цель проекта - учиться JavaScript, но в этом компоненте дополнительно появился SCSS, потому что стили автора написаны не на чистом CSS.

Важное решение: в проекте используем SCSS для стилей, но не подключаем Pug. Повторяющуюся HTML-разметку линейки генерируем через JavaScript-цикл.

Что уже выяснили

  • Браузер напрямую читает index.html, style.css и script.js.
  • SCSS браузер напрямую не читает. Его нужно компилировать в обычный CSS.
  • CodePen автора использует не только SCSS, но и Pug для HTML.
  • Pug-строки, вставленные прямо в обычный HTML, не выполняются.
  • Для 48 повторяющихся элементов линейки не пишем HTML руками, а создаем их через JS.

SCSS в этом компоненте

Sass установлен локально внутри папки components/functional_egg_timer. Такая схема подходит, если SCSS нужен только этому компоненту.

Команды для Sass 
cd components/functional_egg_timer
npm install -D sass

npm list sass
npx sass --version

npx sass style.scss style.css
npx sass --watch style.scss:style.css
Правило работы: пишем в style.scss, а HTML подключает style.css. После запуска Sass-файл style.css лучше не редактировать руками, потому что он будет перезаписываться.

Почему кусок автора не отрабатывал

Этот код выглядит как разметка, но это не HTML. Это Pug-синтаксис, который CodePen сначала компилирует в HTML.

Pug-код из CodePen 
- const faceCount = 48
.egg-top(style=`--face-count: ${faceCount}`)
    .time-ruler-faces-container
        - for (let i = 0; i < faceCount; i++)
            .time-ruler-face-wrapper(style=`--i: ${i}`)
                svg.time-ruler-face(xmlns="http://www.w3.org/2000/svg" viewBox="0 0 1740 76", aria-hidden="true")
                    use(href="#time-ruler")

Pug работает как препроцессор для HTML:

  • .egg-top превращается в <div class="egg-top">.
  • - const faceCount = 48 выполняется на этапе компиляции Pug.
  • for в Pug генерирует 48 HTML-блоков до загрузки страницы в браузере.

Что должен получить браузер

В обычном HTML в итоге должна появиться такая структура. Внутри контейнера должно быть 48 .time-ruler-face-wrapper с индексами от 0 до 47.

Итоговая HTML-идея 
<div class="egg-top" style="--face-count: 48">
    <div class="time-ruler-faces-container">
        <div class="time-ruler-face-wrapper" style="--i: 0">...</div>
        <div class="time-ruler-face-wrapper" style="--i: 1">...</div>
        <div class="time-ruler-face-wrapper" style="--i: 2">...</div>
    </div>
</div>

JS-аналог Pug-цикла

Вместо Pug используем обычный JavaScript: создаем верх яйца, создаем контейнер линейки, генерируем 48 строк HTML и вставляем их в DOM.

Код генерации разметки 
document.addEventListener("DOMContentLoaded", () => {
    const faceCount = 48;

    const egg = document.querySelector(".egg");
    const eggCenter = document.querySelector(".egg-center");

    if (!egg || !eggCenter) {
        return;
    }

    const eggTop = document.createElement("div");
    eggTop.className = "egg-top";
    eggTop.style.setProperty("--face-count", faceCount);

    const facesContainer = document.createElement("div");
    facesContainer.className = "time-ruler-faces-container";

    const facesHtml = Array.from({ length: faceCount }, (_, index) => {
        return `
            <div class="time-ruler-face-wrapper" style="--i: ${index}">
                <svg class="time-ruler-face" xmlns="http://www.w3.org/2000/svg" viewBox="0 0 1740 76" aria-hidden="true">
                    <use href="#time-ruler"></use>
                </svg>
            </div>
        `;
    });

    facesContainer.innerHTML = facesHtml.join("");

    eggTop.append(facesContainer);
    egg.insertBefore(eggTop, eggCenter);
});

Разбор Array.from

Строка ниже создает массив из 48 строк HTML. Название facesHtml лучше читать как "HTML для всех граней".

Array.from 
const facesHtml = Array.from({ length: faceCount }, (_, index) => {
    return `...`;
});
  • { length: faceCount } - объект. Если faceCount = 48, то это как { length: 48 }.
  • Array.from(...) создает массив нужной длины.
  • Вторая часть (_, index) => { ... } запускается для каждого элемента массива.
  • index - номер текущего прохода: 0, 1, 2 ... 47.
  • _ - обычное имя параметра. Его используют как заглушку, когда первый аргумент не нужен.
В callback у Array.from обычно доступны два значения: (currentValue, index). Здесь currentValue не нужен, поэтому вместо него написан _.

Разбор join("")

После Array.from получается массив строк, а не один HTML. Метод join("") склеивает все строки массива в одну большую строку.

join без запятых 
const parts = ["<div>0</div>", "<div>1</div>", "<div>2</div>"];

parts.join("");
// "<div>0</div><div>1</div><div>2</div>"

parts.join();
// "<div>0</div>,<div>1</div>,<div>2</div>"

Важно: join("") не делает один HTML-блок из 48 блоков. Он делает одну текстовую HTML-строку. Потом innerHTML говорит браузеру разобрать эту строку как HTML.

Новые DOM-методы и свойства

setProperty

Устанавливает CSS-свойство через JS. Особенно удобно для CSS-переменных, потому что --face-count нельзя нормально записать через точку.

setProperty 
eggTop.style.setProperty("--face-count", faceCount);

className

Полностью задает строку классов у элемента. Для нового элемента это нормально, потому что старых классов у него еще нет.

className 
eggTop.className = "egg-top";

// Альтернатива:
eggTop.classList.add("egg-top");

append

Кладет один DOM-элемент внутрь другого, в конец родителя. Здесь контейнер с гранями кладется внутрь верхней части яйца.

append 
eggTop.append(facesContainer);

insertBefore

Вставляет новый элемент перед уже существующим элементом внутри одного родителя. Читается так: "внутри egg вставь eggTop перед eggCenter".

insertBefore 
parent.insertBefore(newElement, existingElement);

egg.insertBefore(eggTop, eggCenter);

Итоговая логика блока

  1. Дождаться, пока HTML будет прочитан: DOMContentLoaded.
  2. Найти существующие элементы: .egg и .egg-center.
  3. Создать .egg-top.
  4. Создать .time-ruler-faces-container.
  5. Через Array.from сгенерировать 48 HTML-строк.
  6. Через join("") склеить строки без запятых.
  7. Через innerHTML превратить строку в HTML внутри контейнера.
  8. Через append положить контейнер внутрь eggTop.
  9. Через insertBefore вставить eggTop перед eggCenter.

Что важно помнить дальше

  • SCSS решает задачу стилей, но не генерирует HTML-разметку.
  • Pug мог бы генерировать HTML, но пока его не подключаем, чтобы не усложнять проект вторым препроцессором.
  • Повторяющуюся разметку удобно генерировать через JS, потому что проект учебный и цель - прокачивать JavaScript.
  • Следующий шаг - закончить вставку JS-кода и проверить, что в DOM появилась структура .egg-top с 48 wrapper-элементами.
