Step 1: jQuery-объект, DOM-элемент и проверка через length

На первом JS-шаге подключили jQuery через локальный npm-пакет и начали с самого базового места: как найти элементы на странице и как проверить, что они действительно найдены.

Главная мысль этапа: document.querySelector(...) и $(...) похожи по задаче, но возвращают разные вещи. Из-за этого проверка существования элемента тоже пишется по-разному.

Подключение jQuery

В компоненте jQuery установлен через npm. Но установка пакета сама по себе не делает $ доступным на странице. Для обычной статической HTML-страницы библиотеку нужно подключить тегом script перед своим script.js.

Порядок подключения скриптов 
<script src="./node_modules/jquery/dist/jquery.min.js"></script>
<script src="./script.js"></script>

Порядок важен: сначала загружается jQuery, потом наш файл. Если поменять их местами, в script.js символ $ еще не будет существовать.

Готовность DOM

Запись ниже говорит: выполни код внутри функции после того, как HTML-разметка уже создана и элементы можно искать.

jQuery DOM ready 
$(function () {
    // Здесь уже можно искать элементы.
});

Это короткая jQuery-форма для события готовности документа. В обычном DOM похожая идея выглядела бы через DOMContentLoaded.

Похожая идея на обычном DOM 
document.addEventListener("DOMContentLoaded", () => {
    // Здесь уже можно искать элементы.
});

jQuery-ссылки на элементы

В текущем компоненте основные элементы имеют уникальные id: todoInput, todoAdd, todoList. Их можно найти через jQuery-селекторы.

Поиск элементов через jQuery 
$(function () {
    const todoInput = $("#todoInput");
    const todoAdd = $("#todoAdd");
    const todoList = $("#todoList");

    console.log(todoInput.length, todoAdd.length, todoList.length);
});

В консоли ожидаем 1 1 1. Это значит, что каждый селектор нашел по одному элементу.

В jQuery-проектах часто называют такие переменные с префиксом $: например $todoInput. Это не магия языка, а подсказка человеку: внутри лежит jQuery-объект, а не обычный DOM-элемент.

Обычный DOM: если не найдено, будет null

Когда элемент ищется через document.querySelector, результатом будет либо найденный DOM-элемент, либо null.

Проверка в обычном DOM 
const todoInput = document.querySelector("#todoInput");

if (!todoInput) {
    console.log("Элемент не найден");
    return;
}

Такая проверка работает, потому что null в условии считается ложным значением.

jQuery: если не найдено, будет пустой jQuery-объект

jQuery ведет себя иначе. Даже если селектор ничего не нашел, результатом будет не null, а jQuery-объект с длиной 0.

Неправильная проверка для jQuery 
const todoInput = $("#wrongId");

if (!todoInput) {
    // Этот блок не выполнится.
}

Объект существует, поэтому !todoInput не сработает. Но внутри него нет найденных элементов.

Правильная проверка для jQuery 
const todoInput = $("#todoInput");
const todoAdd = $("#todoAdd");
const todoList = $("#todoList");

if (!todoInput.length || !todoAdd.length || !todoList.length) {
    console.log("Остановись пока остановка не будет последней ☝");
    return;
}
  • $("#todoInput").length === 1 - найден один элемент.
  • $("#wrongId").length === 0 - ничего не найдено.
  • if (!todoInput.length) - нормальная jQuery-проверка на отсутствие элемента.

Итог Step 1

  1. jQuery должен быть подключен до script.js.
  2. $(function () {}) запускает код после готовности DOM.
  3. $("#todoInput") возвращает jQuery-объект, а не обычный DOM-элемент.
  4. Обычный DOM при промахе возвращает null.
  5. jQuery при промахе возвращает пустой объект с length: 0.
  6. Поэтому jQuery-элементы проверяем через .length.

Step 2: Добавление, выполнение, удаление и Enter

На этом этапе todo-list стал рабочим: задача добавляется из input, появляется сверху списка, input очищается, задачу можно отметить выполненной, удалить и добавить по клавише Enter.

Этот шаг начался с ошибки todoText(" "). Она помогла увидеть важное различие: todoText - это строка, а очищать нужно сам input через todoInput.val("").

.val() как getter и setter

Метод .val() работает в двух режимах. Без аргумента он читает значение. С аргументом - записывает новое значение.

Read and clear input 
const todoText = todoInput.val().trim();

if (todoText === "") {
    console.log("Введите текст в поле");
    return;
}

console.log(todoText);
todoInput.val("");
  • todoInput.val() читает значение из input.
  • todoInput.val("") записывает пустую строку в input.
  • todoText(" ") не работает, потому что строка не является функцией.

Создание новой задачи

Новая задача собирается как HTML-строка через template literal. Потом эта строка превращается в jQuery-элемент, чтобы к нему можно было применить jQuery-анимации.

Create todo item 
const newLine = $(`
    <li class="todo-item">
        <span class="todo-text">${todoText}</span>
        <button class="todo-check" type="button" aria-label="Mark task as done">✔</button>
        <button class="todo-delete" type="button" aria-label="Delete task">✖</button>
    </li>
`);

Здесь важно, что newLine - это "что вставляем", а todoList - это "куда вставляем".

Container receives content 
todoList.prepend(newLine);
Важная формулировка: метод вставки вызывается у контейнера, а параметром передается элемент, который нужно вставить. Поэтому не newLine.prepend(), а todoList.prepend(newLine).

Появление новой задачи

Чтобы задача плавно появилась, ее нужно сначала скрыть, потом вставить в DOM, а потом показать. Анимацию вызываем у того элемента, который должен появиться, то есть у новой строки.

Hide, insert, fade in 
newLine.hide();
todoList.prepend(newLine);
newLine.fadeIn("fast");

Модель: создали li в памяти, спрятали его, вставили в список, затем плавно показали. Если вызывать fadeIn у todoList, будет анимироваться контейнер списка, а не новая задача.

Done через делегирование событий

Кнопки задач бывают двух типов: стартовые уже есть в HTML, новые появляются позже через prepend. Поэтому обработчик ставится на постоянный родитель todoList, а конкретная кнопка указывается вторым аргументом.

Delegated done handler 
todoList.on("click", ".todo-check", function () {
    $(this).closest(".todo-item").find(".todo-text").toggleClass("done");
});

Эта цепочка читается как маршрут: кнопка, по которой кликнули, затем ближайший li.todo-item, затем текст внутри этой задачи, затем переключение класса.

Route through the DOM 
$(this)                  // кнопка, по которой кликнули
    .closest(".todo-item") // текущая задача
    .find(".todo-text")    // текст внутри текущей задачи
    .toggleClass("done");  // включить или выключить done

Удаление через fadeOut и remove

Удаление тоже использует делегирование. Сначала находим текущую задачу через closest, потом плавно скрываем ее, а после завершения анимации удаляем из DOM.

Delegated delete handler 
todoList.on("click", ".todo-delete", function () {
    $(this).closest(".todo-item").fadeOut("fast", function () {
        $(this).remove();
    });
});

fadeOut("fast", callback) сначала скрывает элемент с анимацией, затем запускает callback. Внутри callback this указывает на элемент, который только что анимировался.

Добавление по Enter

Финальный кусок логики повторяет поведение кнопки Add при нажатии Enter внутри input. Событие называется keyup, а конкретная клавиша хранится в event.key.

Enter key handler 
todoInput.on("keyup", function (event) {
    if (event.key === "Enter") {
        todoAdd.click();
    }
});
  • .on("keyup", ...) - слушаем момент, когда клавишу отпустили.
  • event.key - читаем, какая именно клавиша была нажата.
  • todoAdd.click() - программно запускаем тот же сценарий, что и клик по кнопке.

Итог Step 2

  1. Input читается через .val(), а очищается через .val("").
  2. Новая задача создается как jQuery-элемент из HTML-строки.
  3. Контейнер todoList получает новую строку через .prepend(newLine).
  4. Новые и старые кнопки работают через делегирование событий.
  5. Done переключается через маршрут this -> closest -> find -> toggleClass.
  6. Удаление использует fadeOut и remove.
  7. Enter вызывает тот же click-сценарий, что и кнопка Add.

jQuery база: методы, цепочки и тип значения

Этот раздел не привязан к одному шагу задачи. Это отдельная памятка по jQuery и методам, чтобы не искать важные языковые моменты внутри рабочего конспекта.

Главный принцип: метод вызывается у значения слева от точки. Поэтому важно понимать, что именно вернул предыдущий вызов: jQuery-объект, DOM-элемент, строку, число или что-то другое.

Почему порядок .val().trim() важен

Рабочая запись выглядит так:

Правильный порядок 
const todoText = todoInput.val().trim();

Эта строка выполняется последовательно слева направо:

  1. todoInput - jQuery-объект, найденный через $("#todoInput").
  2. todoInput.val() читает значение поля формы и возвращает строку.
  3. .trim() вызывается уже у строки и убирает пробелы по краям.
Что происходит по шагам 
todoInput                // jQuery-объект
todoInput.val()          // строка: "   Купить хлеб   "
todoInput.val().trim()   // строка: "Купить хлеб"

Почему todoInput.trim().val() ломается

Ошибочная запись:

Неправильный порядок 
const todoText = todoInput.trim().val();

Здесь JavaScript пытается вызвать .trim() прямо у todoInput. Но todoInput - это jQuery-объект, а trim() является методом строки. Поэтому возникает ошибка:

Ошибка 
Uncaught TypeError: todoInput.trim is not a function
Коротко: trim не умеет работать с input или jQuery-объектом напрямую. Сначала нужен .val(), потому что он достает строку из значения поля.

Цепочка вызовов

Запись вида a.b().c() означает не "вызвать все сразу", а последовательность действий.

Модель цепочки 
const step1 = a.b();
const step2 = step1.c();

Метод .c() вызывается не у исходного a, а у результата a.b(). Поэтому каждый следующий метод должен подходить к типу результата предыдущего метода.

Что именно читает .val()

.val() - jQuery-метод для элементов формы. Он читает или записывает значение свойства value.

Подходит для

  • input
  • textarea
  • select

Не про это

  • текст внутри p
  • текст внутри div
  • текст внутри span
Примеры .val() 
$("#email").val();    // строка из input
$("#message").val();  // строка из textarea
$("#city").val();     // value выбранного option

В todo-list нас интересует именно значение, которое пользователь напечатал в поле:

Todo input 
const todoText = todoInput.val().trim();

.val(), .text(), .html()

Эти методы отвечают за разные части элемента.

Как выбирать метод 
$("#todoInput").val();      // value у input
$(".todo-text").text();     // текст внутри элемента
$(".todo-item").html();     // HTML-разметка внутри элемента
  • .val() - значение формы.
  • .text() - обычный текст внутри элемента.
  • .html() - HTML-разметка внутри элемента.

Frontend-пример

Типичный сценарий: пользователь вводит поисковый запрос, а интерфейс показывает этот запрос в тексте.

Search UI 
<input id="searchInput" placeholder="Search products">
<p id="resultText"></p>
Read value, write text 
const searchInput = $("#searchInput");
const resultText = $("#resultText");

const query = searchInput.val().trim();

resultText.text(`You searched: ${query}`);

Здесь searchInput.val() читает значение формы, trim() очищает строку, а resultText.text(...) записывает обычный текст в параграф.

Методы из todo-list

Ниже короткая справка по jQuery-методам, которые реально используются в компоненте. Это не полный справочник jQuery, а рабочий набор текущей задачи.

.on()

Подписывает элемент на событие. В todo-list используется и напрямую, и как делегирование.

Direct 
todoAdd.on("click", function () {
    ...
});

.on(event, selector, handler)

Делегирует событие: обработчик висит на родителе, но срабатывает для подходящего дочернего элемента.

Delegated 
todoList.on("click", ".todo-check", function () {
    ...
});

.prepend()

Вставляет содержимое в начало контейнера. Метод вызывается у контейнера.

Prepend 
todoList.prepend(newLine);

.hide() / .fadeIn()

hide задает скрытое стартовое состояние, fadeIn плавно показывает элемент.

Appear 
newLine.hide();
todoList.prepend(newLine);
newLine.fadeIn("fast");

.closest()

Идет вверх от текущего элемента и ищет ближайшего родителя по селектору.

Closest 
$(this).closest(".todo-item");

.find()

Ищет дочерний элемент внутри уже найденного контейнера.

Find 
currentItem.find(".todo-text");

.toggleClass()

Переключает класс: если класса нет - добавляет, если есть - убирает.

Toggle class 
currentText.toggleClass("done");

.fadeOut() / .remove()

fadeOut скрывает с анимацией, remove удаляет элемент из DOM.

Remove after animation 
currentItem.fadeOut("fast", function () {
    $(this).remove();
});

.click()

Программно запускает click-событие. В todo-list Enter вызывает тот же сценарий, что и кнопка Add.

Trigger click 
todoAdd.click();

event.key

Это не jQuery-метод, а свойство объекта события. Показывает, какая клавиша была нажата.

Enter check 
if (event.key === "Enter") {
    todoAdd.click();
}

Итог jQuery-базы

  1. Метод вызывается у значения слева от точки.
  2. .val() работает с jQuery-объектом формы и возвращает строку.
  3. .trim() работает со строкой, а не с input.
  4. Цепочка методов зависит от типа результата предыдущего метода.
  5. Для формы используем .val(), для текста элемента - .text(), для разметки - .html().
  6. Для динамических элементов используем делегирование: .on("click", ".selector", handler).
  7. Маршрут по DOM строим через $(this), .closest() и .find().
